# Technical Decisions - GeoJSON Visualizer

**Project**: GeoJSON Traffic Signs Visualizer  
**Last Updated**: 2026-03-03

---

## 1. GeoJSON 数据加载策略

### 决策
支持 URL 参数、网页输入、本地文件上传和拖拽四种方式加载 GeoJSON 数据。

### 理由
- **用户灵活性**：用户可以根据自己的数据存储位置和使用习惯选择最方便的加载方式。
- **避免 Git 大文件**：将 GeoJSON 数据从 Git 仓库中移除，避免仓库膨胀和下载缓慢。
- **即时性**：用户无需部署或修改代码即可使用新数据。
- **隐私**：本地文件处理在浏览器端完成，数据不会上传到任何服务器。

### 替代方案及不选择原因
- **仅支持 URL**：限制了本地文件的使用，不方便离线或内部数据。
- **仅支持文件上传**：不方便分享在线数据，每次都要重新上传。
- **硬编码数据**：完全失去灵活性，每次换数据都需要修改代码并重新部署。

## 2. 地图渲染库选择

### 决策
选择 Leaflet.js 作为地图渲染库。

### 理由
- **轻量级**：Leaflet.js 体积小，加载速度快，非常适合这种简单的可视化应用。
- **API 简洁**：易于上手，提供了丰富的图层和标记控制。
- **生态成熟**：拥有活跃的社区和丰富的插件（尽管本项目未用插件，但未来扩展性强）。
- **兼容性好**：对各类浏览器支持良好。

### 替代方案及不选择原因
- **OpenLayers**：功能强大但相对重量级，对于本项目需求而言有些过度。
- **Mapbox GL JS**：性能优异且支持矢量瓦片，但学习曲线略陡，且免费额度可能受限，本项目追求零成本。
- **Google Maps API**：功能全面但需要 API Key，存在成本和配额限制。

## 3. 性能优化策略 (过滤功能)

### 决策
采用**缓存 Leaflet 图层对象**和**仅显示/隐藏**的策略。

### 理由
- **大幅提升过滤性能**：过滤操作从几秒钟降低到几十毫秒，极大地改善了用户体验。
- **避免重复计算**：Haversine 距离计算和 Layer 对象创建只需在首次数据加载时执行一次，后续过滤操作避免了重复的 CPU 密集型任务。
- **减少 DOM 操作**：不再是每次过滤都删除并重建所有地图元素，而是利用 Leaflet 内部的 `addLayer`/`removeLayer` 机制，减少了浏览器重排和重绘，降低了 DOM 频繁操作带来的性能开销。

### 替代方案及不选择原因
- **每次过滤重新计算并渲染**（旧方案）：导致大数据量下卡顿，用户体验差，已被证明无效。
- **Web Workers 分担计算**：虽然可以进一步优化 Haversine 等计算的性能，但对于本项目而言，缓存机制已足够解决当前痛点，引入 Web Workers 会增加代码复杂度。
- **虚拟滚动 / 分批渲染**：主要解决地图平移缩放时的渲染卡顿，而非过滤时的重新渲染问题。在过滤场景下，缓存是更直接有效的方案。

## 4. 过滤面板交互

### 决策
采用“勾选 + 确认/取消”的批量操作模式，而非实时渲染。

### 理由
- **解决实时渲染卡顿**：大数据量下，每次勾选都触发重新渲染会导致严重卡顿，批量确认避免了这个问题。
- **提升用户体验**：用户可以无缝选择多个过滤条件，只有在确定后才应用更改，提供更好的控制感。
- **避免误操作**：提供“取消”按钮，允许用户放弃不想要的临时选择。

### 替代方案及不选择原因
- **实时渲染**（旧方案）：已被证明在大数据量下性能不可接受。
- **在确认前提供预览**：增加复杂度，且对于本项目而言，确认后立即渲染的性能已足够快，不需要额外的预览步骤。

---

## 5. Segment 过滤逻辑

### 决策
路段 (segment) 仅在包含至少一个**被选中的交通标志类型**时才显示。

### 理由
- **减少视觉噪音**：隐藏没有任何交通标志的路段（即空路段），使地图更清晰，专注于有价值的信息。
- **增强过滤效果**：确保用户在过滤交通标志时，只看到与这些标志直接相关的路段，进一步减少地图上的元素数量，提升性能和信息聚焦度。
- **符合直觉**：用户过滤交通标志时，通常也希望看到这些标志所在的路段，而不是与这些标志无关的整个路网。

### 替代方案及不选择原因
- **显示所有路段，只过滤标志**：导致地图上存在大量空路段，违背了过滤的初衷，无法有效解决视觉混乱问题。
- **只显示路段，不显示标志**：逻辑不符合需求，标志才是主要过滤对象。
